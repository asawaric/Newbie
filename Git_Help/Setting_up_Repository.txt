Setting up a Repository

To store all kinds of your projects in a trackable way one can use the effective way of having one's own repository!

So, how do we set up a Repository?
This page gives the list of most important git commands for setting up your own repository. 
For detail information please read
https://www.atlassian.com/git/tutorials/setting-up-a-repository.

(Note :Checklists are the reminders and revision of your reading and understanding. If you are able to do exercise, that serves the purpose of this page!)

STEP 1: Initialise a new repository 

git init command creates a new repository. It creates a .git subdirectory in the project root, which contains all of the necessary metadata for the repo.You can also convert an existing directory to a git repository. 
============
Checklist
============
git init
git init <directory>
git init --bare <directory>

STEP 2: Clone the git repo 
git clone creates a local copy/copies of your project on a local machine so you can add your changes to it. You can clone with either HTTPs or SSH key. It is aone time operation. 
============
Checklist
============
git clone <repo_name>
git clone <repo_name> <desk_directory>


STEP 3: Configure your repository 
git config configures your git installation. It can define user info to prepferences to the behavior of a repository.
============
Checklist
============
git config
git config --global --edit
git config --global user.name <name>
git config --global user.email <email>
git config --global alias.<alias-name> <git-command>
git config --system core.editor <editor>
